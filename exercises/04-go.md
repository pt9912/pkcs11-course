# Uebung 04 - Go ueber PKCS#11

## Ziel

Du nutzt die native PKCS#11-API aus Go heraus und suchst Slot, Token und Key ueber Attribute statt ueber feste Handles.

## Vorbereitung

```bash
make init-token
make gen-rsa
```

## Aufgabe

1. Starte die vorhandene Go-Demo:
   ```bash
   make go-demo
   ```
2. Lies `lab/go/pkcs11-demo/main.go`.
3. Markiere im Code:
   - Modul-Load
   - `GetSlotList`
   - Token-Label-Match
   - `OpenSession`
   - `Login`
   - Key-Suche per `CKA_ID`
   - `Sign`
4. Pruefe die erzeugten Dateien in `lab/work`.

## Erwartete Ausgabe

- Das Programm nennt den Token `dev-token`.
- Eine Signaturdatei `lab/work/go.sig` entsteht.
- OpenSSL meldet `Verified OK`.
- Session, Logout und Finalize werden auch bei Fehlern aufgeraeumt.

## Fehlerfall

Starte die Demo mit falscher PIN:

```bash
PKCS11_USER_PIN=000000 make go-demo
```

Erwartet: Ein klarer PKCS#11-Fehler wie `CKR_PIN_INCORRECT`.

## Reflexionsfragen

- Warum ist ein Objekt-Handle nur innerhalb seiner Session sinnvoll?
- Warum sucht die Demo den Slot ueber das Token-Label?
