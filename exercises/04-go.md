# Übung 04 — Go über PKCS#11

## Aufgabe

Voraussetzung: Der Token `dev-token` ist initialisiert und der RSA-Key `signing-key` mit ID `01` existiert (`make init-token`, `make gen-rsa`).

Go-Container starten (Default `pkcs11-lab` enthält kein Go):

```bash
docker compose -f lab/docker-compose.yml run --rm pkcs11-go bash
# alternativ pkcs11-dev mit allen Sprachen
```

1. Erstelle ein kleines Go-Programm mit `github.com/miekg/pkcs11`.
2. Lade das Modul aus `PKCS11_MODULE` oder nutze `/usr/lib/softhsm/libsofthsm2.so` als Default.
3. Finde den Slot über das Token-Label `dev-token`, nicht über einen festen Slot-Index.
4. Öffne eine Session, logge dich mit `PKCS11_USER_PIN` ein und suche den privaten Key per `CKA_ID=01`.
5. Signiere die Bytes `hello from go pkcs11` mit `CKM_SHA256_RSA_PKCS`.
6. Exportiere den Public Key mit `pkcs11-tool --read-object --type pubkey --id 01` und verifiziere die Signatur mit OpenSSL.

## Erwartung

- Das Programm bricht sauber ab, wenn Token, Key oder Mechanismus fehlen.
- Die OpenSSL-Verifikation liefert `Verified OK`.
- Der Code ruft `Finalize`, `CloseSession` und `Logout` auch bei Fehlern auf.

## Fehler erzwingen

- Nutze eine falsche PIN. Erwartet: PKCS#11-Fehler `CKR_PIN_INCORRECT`.
- Suche nach `CKA_ID=02`. Erwartet: klare Meldung, dass kein passender RSA-Key gefunden wurde.
