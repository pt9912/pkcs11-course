# Uebung 04 - Go ueber PKCS#11

## Ziel

Du nutzt die native PKCS#11-API aus Go heraus und suchst Slot, Token und Key ueber Attribute statt ueber feste Handles.

## Vorbereitung

```bash
make init-token
make gen-rsa
```

## Aufgabe

1. Starte die vorhandene Go-Demo:
   ```bash
   make go-demo
   ```
2. Lies `lab/go/pkcs11-demo/main.go`.
3. Markiere im Code:
   - Modul-Load
   - `GetSlotList`
   - Token-Label-Match
   - `OpenSession`
   - `Login`
   - Key-Suche per `CKA_ID`
   - `Sign`
4. Pruefe die erzeugten Dateien in `lab/work`.

## Erwartete Ausgabe

- Das Programm nennt den Token `dev-token`.
- `lab/work/go.txt` und `lab/work/go.sig` entstehen.
- OpenSSL meldet `Verified OK`.
- Session, Logout und Finalize werden auch bei Fehlern aufgeraeumt — Teardown-Fehler erscheinen als `Warnung: C_Logout: CKR_...` auf stderr.

## Fehlerfall

`PKCS11_USER_PIN=000000 make go-demo` wuerde schon in der `init-token`/`gen-rsa`-Vorstufe abbrechen. Damit der Fehler tatsaechlich in der Go-Demo auftritt, Vorstufe mit echter PIN laufen lassen und die Demo direkt mit der falschen PIN starten:

```bash
make init-token gen-rsa
docker compose -f lab/docker-compose.yml run --rm \
  -e PKCS11_USER_PIN=000000 \
  pkcs11-go bash -lc 'lab/scripts/13-go-demo.sh'
```

(Im Devcontainer: `make init-token gen-rsa && PKCS11_USER_PIN=000000 lab/scripts/13-go-demo.sh`.)

Erwartet: `C_Login: CKR_PIN_INCORRECT` als Fehler-Ausgabe, Exit-Code 1.
