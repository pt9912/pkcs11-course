# Loesung 04 - Go ueber PKCS#11

## Lauf

```bash
make init-token
make gen-rsa
make go-demo
```

Erwartet:

```text
Token: dev-token
Signatur: /workspace/lab/work/go.sig (...)
Verified OK
```

## Kernablauf im Code

`lab/go/pkcs11-demo/main.go` macht:

1. `pkcs11.New(modulePath)` und `Initialize()`.
2. Slots mit Token ueber `GetSlotList(true)`.
3. `GetTokenInfo(slot)` und Label-Match gegen `dev-token`.
4. `OpenSession` und `Login(CKU_USER, pin)`.
5. Key-Suche mit:
   - `CKA_CLASS = CKO_PRIVATE_KEY`
   - `CKA_ID = []byte{0x01}`
6. Signatur mit `CKM_SHA256_RSA_PKCS`.
7. Public-Key-Export und OpenSSL-Verifikation ueber das Lab-Skript.

## Fehler pruefen

`make`-Targets ueber `gen-rsa`/`init-token` koennen mit veraenderter PIN/Label gar nicht erst durchlaufen. Daher die Vorstufe sauber rendern und nur die Go-Demo mit der Manipulation starten.

Falsche PIN:

```bash
make init-token gen-rsa
docker compose -f lab/docker-compose.yml run --rm \
  -e PKCS11_USER_PIN=000000 \
  pkcs11-go bash -lc 'lab/scripts/13-go-demo.sh'
```

Erwartet: `C_Login: CKR_PIN_INCORRECT`.

Token-Label, das es nicht gibt — Demo direkt aufrufen, damit `init-token` nicht einfach ein neues Token mit diesem Label anlegt:

```bash
make init-token gen-rsa
docker compose -f lab/docker-compose.yml run --rm \
  -e PKCS11_TOKEN_LABEL=missing \
  pkcs11-go bash -lc 'cd lab/go/pkcs11-demo && go run .'
```

Erwartet: klare Meldung, dass kein Slot mit diesem Token-Label gefunden wurde.
